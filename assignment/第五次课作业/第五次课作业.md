### 1. Interest-Aware Graph Contrastive Learning for Recommendation With Diffusion-Based Augmentation
- **作者**：Mengyuan Jing, Yanmin Zhu, Zhaobo Wang, Jiadi Yu, Feilong Tang  
- **期刊**：*IEEE Transactions on Knowledge and Data Engineering*  
- **卷/期/页码**：Vol. 38, No. 1, pp. 414–427  
- **CCF** 等级：A 类
- **DOI**：10.1109/TKDE.2025.3620600  
- **研究问题**：现有图对比学习推荐方法普遍通过随机扰动图结构（删边、删节点）生成对比视图，这类无差别增强会破坏用户行为背后的兴趣一致性，导致对比学习信号无效甚至产生负向作用，无法真正提升表示的泛化性。
- **核心创新贡献**：
	1.首次将扩散模型引入图对比推荐的数据增强过程，利用扩散过程的逐步加噪-去噪机制生成语义丰富且可控的对比视图。
	2.设计兴趣感知的增强策略，在扩散重建时约束去噪方向与用户多兴趣子图保持一致，避免增强视图偏离核心偏好模式。
	3.利用扩散过程天然的不确定性建模能力，使模型能够区分真实交互与噪声交互，从而缓解交互数据中的噪声干扰。
- **关键词**：Graph-based recommendation, contrastive learning, diffusion model.

### 2. Knowledge-Enhanced Graph Contrastive Learning for Recommendations
- **作者**：Xiaofeng Wang, Zheng Zhang, Yuanyuan Qi, Guodong Shen, Shuaiming Lai, Yuntao Chen, Fang Zhou, Daying Quan  
- **期刊**：*IEEE Transactions on Multimedia*  
- **卷/期/页码**：Vol. 28, pp. 684–699  
- **CCF** 等级：A 类
- **DOI**：10.1109/TMM.2025.3626976  
- **研究问题**：现有图对比推荐大多依赖用户-物品交互图的结构扰动，在长尾推荐场景下提升有限，同时还忽略了知识图谱中实体与关系的丰富语义。  
- **核心创新贡献**：
	1.提出知识增强的图对比学习框架 KECL，采用知识图谱嵌入增强代替图结构增强构建对比视图，无需多次图卷积前向传播，大幅降低训练开销。
	2.在表示层面执行知识感知的增强操作，实现高效的表示级对比学习，在长尾物品、冷启动用户场景下性能优势明显。
- **关键词**：Recommendation system, knowledge graph, contrastive learning, data augmentation.

### 3. XSimGCL: Towards Extremely Simple Graph Contrastive Learning for Recommendation
- **作者**：Junliang Yu, Xin Xia, Tong Chen, Lizhen Cui, Nguyen Quoc Viet Hung, Hongzhi Yin  
- **期刊**：*IEEE Transactions on Knowledge and Data Engineering*  
- **卷/期/页码**：Vol. 36, No. 2, pp. 913–926  
- **CCF** 等级：A 类
- **DOI**：10.1109/TKDE.2023.3288135  
- **研究问题**：现有图对比学习推荐方法依赖复杂的图结构增强，训练开销大、工程实现繁琐；且图增强本身对推荐任务的增益有限，甚至会引入噪声损害节点表示的质量，性价比极低。
- **核心创新贡献**：
	1.提出极简图对比学习范式，仅通过在嵌入空间添加可控随机噪声来构建对比视图，彻底弃用图结构扰动。
	2.理论上证明嵌入噪声等价于一种隐式数据增强，能有效缓解表示坍塌，且无需调节增广比例等敏感超参数。
	3.在多个数据集上验证，尽管方法极简，但推荐精度和训练效率均大幅超越所有基于图增强的同类方法。
- **关键词**：Contrastive learning, data augmentation, recommendation, self-supervised learning.

### 4. Knowledge Graph Pruning for Recommendation
- **作者**：Xi Zhu 等 
- **期刊**：*ACM Transactions on Information Systems*  
- **卷/期/页码**：Vol. 44, Issue 1  
- **CCF** 等级：A 类
- **DOI**：10.1145/3769107  
- **研究问题**：  
  知识图谱增强推荐通常将整张 KG 嵌入模型，但大部分三元组与推荐任务无关，庞大的知识图谱带来极高的计算与存储成本，限制了复杂模型的工业落地；且大量冗余、无关的知识节点会引入噪声，反而损害推荐性能。
- **核心创新贡献**：
	1。提出面向推荐任务的专属知识图谱剪枝方法 KGTrimmer，在最小化推荐性能损失的前提下，精准移除无价值的知识节点，大幅降低计算开销。
	2.设计双视角节点重要性评估器：集体视角基于用户协同信号提取社区共识，整体视角基于节点固有属性与全局流行度学习全局掩码，全面衡量节点价值。
	3.剪枝后的稀疏子图大幅降低 GNN 消息传递开销，可作为插件适配现有多种 KG 推荐模型，在精度无损或微升的前提下显著加速训练。

## 三、精读对象——XSimGCL: Towards Extremely Simple Graph Contrastive Learning for Recommendation

### 1、选择理由
- XSimGCL舍弃了此前图对比推荐中所有复杂的图增广手段（边丢弃、节点丢弃、随机游走），仅通过在嵌入空间添加可控随机噪声来构建对比视图，方法极简但效果很好。
- 论文背景涉及协同过滤（LightGCN）、图对比学习（SGL）、对比损失（InfoNCE）等关键概念，覆盖了图推荐领域最核心的几块拼图，精读这一篇可以自然串联起完整知识链。
- 论文给出了关键的理论证明：嵌入扰动等价于一种隐式数据增强，能缓解表示坍缩。同时，作者提供了开源模型，代码结构清晰，非常利于复现和修改。

### 2、目标问题
- 为什么给嵌入加噪声，就等价于做图增广？
- “表示坍缩”在图推荐里到底长什么样？XSimGCL 又是如何避免的？
- 它和 SGL、SimGCL 的区别在哪，为什么丢掉东西反而变强，效果变得更好？
