# 图推荐

## 一、方向基本概述

图推荐（Graph-based Recommendation）是推荐系统与图神经网络结合的核心研究方向，核心思想是将用户、物品及交互关系抽象为图结构数据，通过图神经网络的消息传递与高阶邻居聚合机制，挖掘用户 - 物品之间的深层协同关联，有效缓解传统协同过滤的数据稀疏与泛化能力不足问题。

##  二、核心术语

| 英文术语                         | 中文翻译       | 简要说明                                   |
| ---------------------------- | ---------- | -------------------------------------- |
| Collaborative Filtering (CF) | 协同过滤       | 利用用户-物品交互历史进行推荐的基本范式                   |
| User-Item Bipartite Graph    | 用户-物品二部图   | 图推荐的标准数据结构，节点分为用户、物品两类，边代表交互行为         |
| Graph Neural Network (GNN)   | 图神经网络      | 在图结构上传递、聚合邻居信息的深度模型                    |
| Message Passing              | 消息传递       | 图神经网络的核心运算，节点通过聚合邻居节点的特征信息更新自身表征       |
| Embedding Propagation        | 嵌入传播       | GNN 中逐层聚合邻居嵌入的过程，LightGCN 的核心操作        |
| Contrastive Learning (CL)    | 对比学习       | 拉近正样本对、推远负样本对，学习判别性表示                  |
| Data Augmentation (Graph)    | 图数据增强      | 对图结构进行扰动生成多视图                          |
| Over-smoothing               | 过平滑        | 图推荐的经典问题，指堆叠过多图卷积层后，不同节点的表征趋于一致，模型效果下降 |

## 三、方法分类

### 3.1 LightGCN
- **核心思想**：移除传统 GCN 中的非线性激活和特征变换矩阵，仅保留归一化的邻域聚合与多层层组合，用“轻量级”图传播学习用户与物品的嵌入。

### 3.2 SGL、SimGCL
- **核心思想**：在图协同过滤的基础上，通过结构扰动构造多个视图，利用对比损失最大化不同视图间同一节点表示的一致性，注入自监督信号来改善长尾推荐和表示鲁棒性。

##  四、任务与数据形式

**核心任务**：给定用户的历史交互记录，为用户预测并排序最可能感兴趣的 N 个物品
**数据形式**：用户-物品二分图  
- 用户节点集合 U、物品节点集合 I、交互边集合 E，边可附带权重
- 典型数据集：MovieLens、Gowalla, Yelp2018, Amazon-Book
- 数据划分：Leave-one-out，即每个用户的最后一次交互作为测试集，倒数第二次作为验证集，其余为训练集
- 输入格式：user_id、item_id交互记录文件，图结构由交互直接构建邻接矩阵

##  五、评价指标

- **1、HR@K**
测试集中用户的真实交互物品出现在推荐列表 Top-K 中的比例。

- **2、NDCG@K）**
综合考虑推荐物品的排序位置，对排名靠前的正确结果赋予更高权重。

- **3、Recall@K**
用户真实交互的所有物品中，被推荐列表 Top-K 覆盖的比例。

- **4、Precision@K**
推荐列表 Top-K 中，用户真实交互物品的占比。

##  七、问题

**问题**：在规模较大的推荐场景中，全量用户-物品二部图的节点规模极大，基于全图消息传递的训练是否计算量极大，导致过多的消耗；当下普遍采用哪些优化方案，这些方案在减少计算量的同时，是否会对模型的效果造成折损？
